# 长离 - 需求文档

## 项目信息

- **项目名称**: 长离 (ChangLi)
- **版本**: v1.0.0
- **日期**: 2026-06-10
- **类型**: 桌面应用

## 1. 项目背景

### 1.1 业务背景
用户需要一个全功能的视频管理播放平台，能够从多个网站拉取资源、自动下载磁力链接、管理本地视频库，并提供高质量的播放体验。特别针对动漫资源管理，支持演员关联、多标签分类、观看进度追踪等功能。

### 1.2 技术背景
- 需要跨平台支持（macOS + Windows）
- 需要处理多个不同结构的网站
- 需要高质量视频播放（原画质、原帧率）
- 需要稳定的下载管理
- 需要复杂的数据关系管理（资源-演员-作品）

## 2. 功能需求

### 2.1 网站资源拉取模块

#### F-2.1.1 网站配置管理（模块化）
- **描述**: 用户可以添加、编辑、删除网站配置，支持模块化配置以兼容不同网站
- **优先级**: 高
- **验收标准**:
  - [ ] 支持模块化配置，强兼容性
  - [ ] 不同网站可配置不同的解析规则
  - [ ] 支持配置 CSS 选择器提取标题、链接、元数据
  - [ ] 支持启用/禁用网站
  - [ ] 支持导入/导出网站配置
  - [ ] 支持在线播放能力标记（网站是否支持在线播放）

#### F-2.1.2 资源搜索
- **描述**: 跨网站搜索资源
- **优先级**: 高
- **验收标准**:
  - [ ] 支持关键词搜索
  - [ ] 支持选择搜索哪些网站
  - [ ] 展示搜索结果列表（标题、来源、磁力状态）
  - [ ] 支持分页加载

#### F-2.1.3 资源详情
- **描述**: 查看资源详细信息

  - [ ] 支持一键复制磁力链接

#### F-2.1.4 资源订阅

## 3. 非功能需求

### 3.1 性能需求
- **启动时间**: < 3 秒
- **搜索响应**: < 2 秒
- **播放启动**: < 1 秒
- **内存占用**: < 500MB

### 3.2 兼容性需求
- **操作系统**: macOS 12+, Windows 10+
- **视频格式**: MP4, MKV, AVI, FLV, MOV, WMV, WebM
- **字幕格式**: SRT, ASS, SSA, VTT

### 3.3 安全需求
- 网站配置本地存储
- 下载文件本地存储
- 无用户数据上传

## 4. 技术约束

### 4.1 技术栈
- **桌面框架**: Tauri 2.0
- **前端**: React + TypeScript
- **播放器**: mpv
- **下载**: aria2
- **数据库**: SQLite

### 4.2 依赖
- Node.js >= 18
- Rust >= 1.70
- aria2（用户安装）

## 5. 数据关系

### 5.1 核心实体关系
```
资源 (Resource)
├── 标题
├── 海报
├── 简介
├── 标签 (多对多)
├── 分类 (用户自定义)
├── 状态 (已完结/连载中)
├── 更新频率
├── 更新日期
├── 演员 (多对多)
└── 集数 (一对多)
    ├── 第1集
    ├── 第2集
    └── ...

演员 (Actor)
├── 姓名
├── 海报/写真
├── 简介
└── 参演作品 (多对多)

标签 (Tag)
├── 名称
└── 资源 (多对多)
```

### 5.2 观看进度
```
观看记录 (WatchProgress)
├── 资源ID
├── 集数
├── 观看位置 (分钟)
├── 总时长
└── 更新时间
```
