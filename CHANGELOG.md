# Changelog

## [1.9.20] - 2026-07-13

### Fixed
- **返回上一页位置**: 修复从详情页返回列表时恢复位置不准确的问题，筛选/搜索参数变化不再触发滚动重置
- **首页/视频页海报**: 兼容历史海报缓存和失效路径，卡片优先使用可显示的缓存兜底，避免被占位图替代
- **订阅管理版本展示**: 订阅管理列表直接展示当前订阅版本，方便确认订阅选择

## [1.9.19] - 2026-07-13

### Added
- **订阅版本展示**: 视频集订阅管理条直接显示当前订阅版本，长版本名支持悬停查看完整内容

### Fixed
- **首页/视频页海报**: 历史数据只有海报缓存、没有海报路径时，列表按需返回缓存兜底，避免卡片显示占位图
- **返回演员页位置**: 修复返回时滚动位置保存 key 被新页面提前覆盖，避免演员页恢复到最底部
- **订阅重复更新**: 支持 Mikanani `[01][1080P]` 这类标题格式识别集数，避免本地已有第 1 集仍提示更新
- **标签弹窗体验**: 标签类型和关联分类选择改为更清晰的气泡式交互

## [1.9.18] - 2026-07-12

### Added
- **标签管理页视觉优化**: 标签页增加概览卡片、分组面板和更清晰的标签卡片层级
- **自动内存清理**: 主窗口和播放器窗口增加定时/后台/内存压力清理，释放页面缓存、预览图缓存和 mpv 可重建缓冲，降低长时间使用的内存堆积

### Fixed
- **播放器实时缩放**: 右下角拖拽改为自定义等比实时缩放，使用 RAF 节流和队列保护，兼顾跟手体验与稳定性
- **批量更新海报**: 批量更新入口改为后台完整队列，先修复缺失海报再静默重建海报缓存，不阻塞用户操作
- **列表性能**: 列表刷新不再传输大海报缓存，避免海报缓存导致卡顿
- **海报缓存尺寸**: 新生成的海报缓存从 1200px 降到 600px，降低保存和批量更新后的处理负担
- **保存修改卡顿**: 视频集保存、状态切换、追番/已看、分类切换、删除等操作不再默认触发整库刷新，改为本地即时更新
- **返回滚动位置**: 返回列表时按完整路径和筛选参数恢复滚动位置，并在内容加载后重试恢复
- **播放器切集稳定性**: 串行化 mpv 命令，减少切集时 IPC 并发导致的崩溃风险
- **更新安装提示**: 下载完成后在原弹窗内提示立即安装或暂不安装

## [1.9.17] - 2026-07-12

### Added
- **标签识别**: 标签卡片使用稳定彩色背景，区分更直观
- **订阅阅读**: 每个订阅使用稳定彩色背景，列表更容易扫读

### Fixed
- **订阅检查更新重复提示**: 检查结果只按当前 RSS 和本地视频集实时判断，不再受已知 GUID 或历史下载记录干扰
- **订阅版本匹配**: 画质、语言、来源、容器统一变体匹配，2160p 可匹配 4k，简繁内封等可匹配简繁
- **订阅季集识别**: 条目无季号时从 RSS 标题识别季号，全局集数按本地已有季结构动态换算，避免第三季第 1 集重复提示
- **订阅更新列表**: 新增条目分组去掉绿色背景，改为中性白底
- **视频页返回体验**: 返回上一页时恢复原滚动位置
- **海报展示**: 新增季检查优先使用季文件夹海报，列表卡片恢复海报缓存展示

## [1.9.16] - 2026-07-12

### Added
- **订阅编辑功能**: 每个订阅新增编辑按钮，可修改关联视频集和版本

### Fixed
- **启动/列表卡顿**: get_video_series_list不调用series_from_row避免读文件转base64，784视频集从卡1分钟变为秒开
- **选集删除卡顿**: 批量删除用单次IPC替代逐个调用
- **选集编辑按钮**: "批量删除"改为"编辑"
- **订阅检查更新匹配失败**: 统一过滤逻辑为prefix_part_matches_title，支持简中/繁中/简繁完整变体映射（简日内嵌、简体内嵌、简繁内封等）
- **参演作品海报不显示**: 空视频集和有视频的作品都从video_series单独读取poster_base64，不依赖SQL别名
- **滚动恢复失效**: 去掉navigationType依赖，兼容所有返回场景
