# Changelog

## v1.7.11 (2026-07-01)

### 新增
- 合并 PR #10：统一 ChangLi 桌面端视觉系统，新增 PageMotion 页面动效组件
- 引入 GSAP / @gsap/react，用于页面级细微动效

### 优化
- 首页、视频库、搜索、下载、标签、演员、详情页、设置页和播放器统一卡片、面板、弹窗、按钮、输入框、右键菜单和空状态样式
- 保留 PR #10 原始 UI 升级设计，不额外改变视觉决策
- 保留 v1.7.10 的通知右侧滑入、演员海报同步、参演作品标题 hover 和更新确认弹窗修复

## v1.7.10 (2026-06-30)

### 修复
- 通知恢复为历史右侧滑入渐入样式，去掉绿色线条，统一为简洁黑底提示
- 新增/删除/设主演员海报后同步刷新演员页、演员列表和首页热门演员，第一张海报立即可见
- 参演作品标题保持最多两行省略，鼠标停留时可查看完整名称
- 检查更新发现新版时先弹窗确认，用户选择后再跳转下载安装包

## v1.7.9 (2026-06-30)

### 修复
- Toast 通知位置调整到右上角设置图标下方，并从右上角渐入
- 首页热门演员卡片恢复原始大尺寸，新增数量通过横向滚动展示，不再缩小卡片

## v1.7.8 (2026-06-30)

### 改进
- 设置页检查更新改为 GitHub Release API：不再依赖 Tauri updater 的 `latest.json`/`.sig`
- 检测到新版本时自动打开最新 Windows 安装包下载链接，用户按安装器完成更新
- 移除 Tauri updater 插件和权限残留，避免发布链路再被 updater 签名产物卡住

## v1.7.7 (2026-06-30)

### 新增
- 合并 PR #7：演员详情页访问次数统计，演员列表按热门度排序
- 首页热门演员扩展为 10 个并支持横向滚动、箭头切换和查看更多
- 视频库默认展示排序第一的大类，不再硬编码 anime
- 演员参演作品名称支持 hover 查看完整标题

### 发布
- 暂时撤掉失败的 Tauri updater JSON 自动上传步骤，保证 Windows exe/msi 发布链路先恢复稳定；后续改为 GitHub Release API 下载更新方案

## v1.7.6 (2026-06-30)

- 修复设置页检查更新因 Release 缺少 updater JSON/签名导致无法获取有效更新信息的问题

### 优化
- 设置页当前版本与版本更新记录改为构建时自动生成，避免手写遗漏
